# Project Architecture, Patterns, and Decisions Summary

## 🏗️ System Architecture
The **fwber** platform is a modern, privacy-first proximity social network organized as a TypeScript monorepo. It transitions the original PHP-based logic into a high-concurrency stack designed for real-world serendipity and decentralized interaction.

### Core Stack
*   **Backend (`fwber-backend-ts`)**: Node.js (ESM) with Express.js. It uses **Prisma ORM** for type-safe database interactions (primarily MySQL).
*   **Frontend (`fwber-frontend`)**: Next.js 15+ (React 19) utilizing the App Router, TailwindCSS for styling, and Lucide for iconography.
*   **Real-time Layer**: Socket.io handles bi-directional events like chat, notifications, and the "Local Pulse."
*   **Federation Layer**: A complete **ActivityPub** implementation allowing the platform to interact with the wider Fediverse (Mastodon, Pleroma, etc.).

## 🌐 Federation & ActivityPub Implementation
Phase 9 "Social Velocity & Federation" has established a robust, interactive bridge to the decentralized web.

### Key Components
*   **WebFinger**: Implemented via `FederationService.resolveWebFinger`, allowing handle-to-URI resolution (e.g., `@user@domain.com` -> actor JSON).
*   **Outbound Delivery**: Uses RSA-SHA256 HTTP Signatures to sign and deliver activities (Follow, Like, Announce, Create, Update) to remote inboxes.
*   **Inbound Processing**: Processes Activities asynchronously. `handleCreate` manages incoming remote posts and detects `@mentions` for real-time notifications.
*   **Persistence**: Tracks all outbound interactions in the `federation_outbox` and inbound activities in `federation_inbox`.

## 🛡️ Security Patterns
Security is a foundational pillar, not an afterthought.
*   **SSRF Protection**: Centralized in `lib/ssrf.ts`. It performs DNS resolution of outbound federation targets to ensure they do not resolve to private, loopback, or reserved IP ranges (e.g., 10.x, 127.x, 172.16-31.x, 192.168.x).
*   **XSS Sanitization**: All untrusted remote federated content is sanitized in the frontend using **DOMPurify** before being rendered via `dangerouslySetInnerHTML`.
*   **Actor Isolation**: Federated interactions are strictly isolated from local proximity data to prevent leaking user coordinates to remote servers.

## 🛠️ Architectural Patterns & Decisions
*   **Service-Oriented Logic**: Business logic is encapsulated in `*Service.ts` classes (e.g., `FederationService`, `MatchingHeuristicService`, `ActivityNotificationService`), keeping Express routes clean and focused on I/O.
*   **Environment Portability**: The system uses `API_DOMAIN` and `FRONTEND_URL` environment variables to construct canonical URIs, ensuring the platform can be easily deployed across dev, staging, and production environments.
*   **Autonomous Protocol**: Every critical service is instrumented with the `AutonomousTaskExecutor` to log system-level tasks, facilitating real-time monitoring and self-healing.
*   **Data Serialization**: A global `serialize` utility handles BigInt-to-Number conversion for JSON responses, preventing precision loss in the frontend.
*   **Geometric-Mean Matching**: The matching engine uses an OkCupid-style geometric mean satisfaction heuristic, weighting philosophical values (80%) against real-time physical proximity (20%).

Users can now search for remote actors, manage follow relationships, like/boost/reply to remote posts, and receive real-time notifications for mentions, all while protected by robust security layers.

- **Unified TypeScript Stack**: The project has migrated from a legacy PHP/Laravel environment to a modern, unified TypeScript architecture.
    - **Backend (`fwber-backend-ts`)**: Node.js (ESM) using Express.js.
    - **Frontend (`fwber-frontend`)**: Next.js 15 (React 19) with Tailwind CSS and Framer Motion.
- **Service-Oriented Business Logic**: Core domains are encapsulated in `src/services/` to ensure modularity and high testability. Key services include:
    - `FederationService`: Manages ActivityPub lifecycles and peer interactions.
    - `WingmanService`: Handles conversation analysis and proactive social nudges.
    - `MatchingHeuristicService`: Implements sophisticated compatibility algorithms.
    - `ActivityNotificationService`: Centralized engine for notifications and Socket.io broadcasts.
- **Data Layer Governance**: Prisma ORM with snake_case field naming. `BigInt` primary keys are standardized for scalability, with a global `serialize` utility in `lib/prisma.ts` converting them to `Strings` for safe JSON transport to the frontend.
- **Real-Time Integration**: Socket.io facilitates bidirectional communication for messaging, typing indicators, and system-wide "Local Pulse" updates.

## 🌐 Decentralization & Federation (Phase 9)
- **ActivityPub Interactive Layer**: fwber is now a secure, interactive Fediverse node. It supports the full lifecycle of decentralized interactions: `Follow`, `Accept`, `Like`, `Announce` (Boost), `Create` (Posts/Replies), `Update` (Profiles), `Delete`, and `Undo`.
- **Identity Resolution**: Native handle-to-URI resolution using **WebFinger** (e.g., `@user@domain.me`) integrated into search and discovery flows.
- **Security-First Federation**:
    - **Anti-SSRF & DNS Rebinding**: Outbound federation requests use a custom HTTPS agent (`lib/ssrf.ts`) that resolves hostnames once and **pins the validated IP address**, preventing Time-of-Check to Time-of-Use (TOCTOU) risks.
    - **Isomorphic XSS Protection**: All untrusted remote content (bios, post content) is sanitized via `isomorphic-dompurify` on both the server (SSR) and client.
    - **Data Isolation**: Internal API feeds (`/activity`, `/posts`) are strictly secured via `authenticate` middleware and filtered by the user's target actor URI to prevent privacy leaks.

## 🤖 AI & Intelligent "Wingman" Systems
- **Multi-Provider Resilience**: AI features implement a robust failover sequence in `wingman-ai.ts`: Primary (NVIDIA NIM) -> Fallback (OpenRouter/Gemini) -> Last Resort (OpenAI).
- **Proactive AI Wingman**: A specialized service (`WingmanService.ts`) monitors messaging flows and autonomously generates "social nudges." These suggest real-world actions (venue check-ins, date invites) via the notification system when natural conversation transitions are detected.
- **Narrative Compatibility (v2.1.9)**: Uses AI to generate atmosphere-rich "Cyber-Noir" reports explaining the specific values and differences between users, replacing pure numerical percentages with human-readable insights.

## 🛡️ Patterns & Principles
- **Autonomous Execution Protocol**: AI agents are instrumented via the `AutonomousTaskExecutor`, logging all maintenance and feature tasks into a persistent ledger (`autonomous_actions`) for real-time monitoring via the Admin Dashboard.
- **Compatibility Heuristics**: Employs an OkCupid-style geometric-mean satisfaction formula: `(Satisfaction A * Satisfaction B) ^ (1/2)`. Scores are weighted 80/20 between value-based questions and real-time proximity signals.
- **Documentation Governance**: Continuity is maintained via a core documentation suite: `VISION.md`, `ROADMAP.md`, `TODO.md`, `CHANGELOG.md`, `MEMORY.md`, and `HANDOFF.md`.

**Production Status:** Phase 9 COMPLETE. All security hardening, interactive federation loops, and proactive AI tools are implemented, verified, and wired to the UI. Version 2.2.9 is established as the new production baseline.

- **Security Hardening:**

## 🏗️ Core Architecture
The project is a high-performance, hyper-local social platform transitioning from a legacy PHP/Rust foundation into a **Unified TypeScript Monorepo**.

### Backend (`fwber-backend-ts`)
- **Runtime:** Node.js (ESM).
- **Framework:** Express.js with a service-oriented architecture.
- **ORM:** Prisma for type-safe database access (MySQL/MariaDB).
- **Key Services:**
    - `SentimentAnalysisService`: Multi-provider AI engine (NVIDIA NIM -> Fallback) that detects user and neighborhood moods.
    - `WingmanService`: Proactive AI subagent that analyzes message streams to push social "nudges" (date invites, venue suggestions).
    - `FederationService`: Full ActivityPub implementation supporting signed social loops (Follow, Like, Boost, Reply, Undo).
    - `MatchingHeuristicService`: Employs an OkCupid-style geometric-mean compatibility formula `(SatA * SatB)^(1/2)`, weighted 80/20 between personality and proximity.
    - `GeoScreenerService`: Hybrid geospatial indexing using H3-js and Redis-based Bloom filters for O(1) empty-cell rejection.
    - `MaintenanceService`: The self-healing backbone that handles system heartbeats, sentiment refreshing, and autonomous vibe nudging.

### Frontend (`fwber-frontend`)
- **Framework:** Next.js 15 (App Router).
- **Styling:** Tailwind CSS with a "Cyber-Noir" aesthetic.
- **Identity UI:** `EvolvingAvatar` leverages Framer Motion and CSS "Auras" to reflect the user's live emotional state.
- **Security:** Standardized on `isomorphic-dompurify` for sanitizing untrusted federated content.

## 🛠 Design Patterns & Decisions
- **Autonomous Execution Protocol:** All critical logic is instrumented via `AutonomousTaskExecutor`. Every major action is logged to the `autonomous_actions` ledger, enabling real-time monitoring and automated behavior adjustments (e.g., toggling `strict_mode` based on failure rates).
- **Security Hardening:**
    - **Mitigating DNS Rebinding:** Outbound federation requests use a custom HTTPS agent (`lib/ssrf.ts`) that resolves hostnames once and pins the resulting IP to prevent TOCTOU attacks.
    - **XSS Protection:** Enforced sanitization at both the API proxy and component level.
- **Dynamic Identity:** A core design choice that user identity is fluid. `current_emotion` is updated autonomously by AI based on social interaction, altering both visual aura effects and AI avatar generation prompts.
- **Threaded Community Forum:** Evolved the "Local Pulse" from a simple feed into a structured community board with topic-based filtering and support for recursive threaded conversations.
- **Vibe-Matched Economy:** Integrated AI sentiment analysis with merchant promotions, allowing for autonomous "Vibe Nudges" where the system triggers deals when the neighborhood mood aligns with specific merchant offers.

## 📈 Evolution Milestones
- **v2.1.x:** Value-based matching engine and narrative reports.
- **v2.2.x:** Secure ActivityPub federation and social velocity.
- **v2.3.x:** Intelligence, Sentiment, and AI-Driven Local Economy.

# fwber Project Memory: Architecture, Patterns & Decisions

## 🏗️ System Architecture
The project is a high-fidelity social platform organized as a **Unified TypeScript Monorepo**, encompassing a migration from legacy PHP/Rust into a modern, performance-oriented TS stack.

### Backend (`fwber-backend-ts`)
- **Framework:** Node.js (ESM) using Express.js.
- **Data Layer:** Prisma ORM with MySQL/MariaDB.
- **Services Architecture:**
- **Services Architecture:**
    - **Intelligence:** `SentimentAnalysisService` (multi-provider AI mood detection), `WingmanService` (proactive conversation nudges), `NarrativeService` (AI-generated compatibility reports).
    - **Social Graph:** `FederationService` (Full ActivityPub implementation with signed Follow/Like/Boost/Undo social loops).
    - **Matching:** `MatchingHeuristicService` (geometric-mean algorithm `(SatA * SatB)^(1/2)` weighted 80/20 against proximity).
    - **Geospatial:** `GeoScreenerService` (Hybrid H3-js local indexing + external Rust service bridge with Redis Bloom filters for O(1) empty-cell rejection).
    - **Infrastructure:** `TokenDistributionService` (FWB tokenomics), `MaintenanceService` (Autonomous heartbeat and self-healing logic).

### Frontend (`fwber-frontend`)
- **Framework:** Next.js 15 (App Router).
- **Styling:** Tailwind CSS with a distinctive "Cyber-Noir" aesthetic.
- **Key Components:** `EvolvingAvatar` (Framer Motion + CSS Auras reflecting emotional state), `LocalPulse` (Threaded community forum), Merchant Portal (Vibe Intelligence analytics).

## 🛠 Architectural Patterns
- **Autonomous Execution Protocol:** A system-wide directive where critical operations are instrumented via `AutonomousTaskExecutor`. Actions are logged to a persistent `autonomous_actions` ledger, enabling real-time monitoring and autonomous behavior adjustment (e.g., toggling `strict_mode` if failure rates exceed 20%).
- **AI Multi-Provider Failover:** To ensure resilience, AI features follow a tiered provider logic: **NVIDIA NIM** (Primary) -> **OpenRouter/Fallback** -> **OpenAI** (Last Resort).
- **Security standard:**
- **Security standard:**
    - **SSRF Hardening:** Outbound federation requests use a custom HTTPS agent (`src/lib/ssrf.ts`) that performs DNS resolution once and **pins the IP address** to mitigate DNS Rebinding (TOCTOU) attacks.
    - **XSS Protection:** `isomorphic-dompurify` is used for both server-side and client-side sanitization of untrusted federated content.
- **ESM Standard:** The backend is strictly ESM, requiring specific Jest configurations (`--experimental-vm-modules`) and `BigInt` serialization handling.

## ⚖️ Key Design Decisions
- **Identity Fluidity:** Rejected static user profiles in favor of dynamic identities. `current_emotion` is autonomously updated by AI based on social activity, which in turn alters the user's visual Aura and the prompts used for AI-generated avatars.
- **Local-First Commerce:** Integrated AI sentiment analysis into the merchant layer. This enables "Autonomous Vibe Nudging," where merchant deals are triggered to nearby users when the neighborhood mood aligns with specific offer targets.
- **Proximity-Focused Sociality:** Prioritized the "Local Pulse" as a hyper-local community layer rather than a global feed, supporting recursive threaded conversations and topic-based neighborhood organization.
- **Transparent Matching:** Decided to provide AI-powered narrative explanations for match scores to build user trust beyond raw numerical percentages.
